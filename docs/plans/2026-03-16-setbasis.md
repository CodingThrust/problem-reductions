**Goal:** Add the decision problem `SetBasis` to the library, CLI, example database, trait-consistency checks, and paper, using the issue’s `k * universe_size` binary encoding and the issue’s worked example as the canonical example.

**Architecture:** `SetBasis` is a satisfaction problem over `k` basis rows, each row encoding a subset of the universe. A configuration is valid iff for every target set in the collection, the union of all encoded basis sets that are themselves subsets of that target equals the target. This avoids an exponential subset-search inside `evaluate()` while preserving the issue’s semantics and binary encoding.

**Tech Stack:** Rust library + serde + inventory registry + CLI `pred create` + Typst paper + cargo tests/clippy

- Issue comments to preserve:
  - use `universe_size`, not `set_size`
  - use `vec![2; k * universe_size]`, not exponential candidate-subset variables
