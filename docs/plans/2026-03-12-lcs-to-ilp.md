## Step 2: Implement LCS to ILP reduction

### 2a. Create `src/rules/longestcommonsubsequence_ilp.rs`

Following the match-pair formulation from Blum et al. 2021:

**Variable mapping (2-string case):**
For each pair (j1, j2) where s1[j1] == s2[j2], create binary variable m_{j1,j2}.

**Constraints:**
1. Each position in s1 matched at most once
2. Each position in s2 matched at most once
3. Order preservation (no crossings): for (j1,j2),(j1',j2') with j1<j1' and j2>j2': m_{j1,j2} + m_{j1',j2'} <= 1

**Objective:** maximize sum of all match variables

**Solution extraction:** matched positions with m=1, sorted by j1, give LCS characters.

### 2b. Register in `src/rules/mod.rs`

Add `#[cfg(feature = "ilp-solver")] mod longestcommonsubsequence_ilp;`

### 2c. Overhead
